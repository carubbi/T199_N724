# Orientação de Entrega no DreamShaper para o T1

**Prazo final desta etapa:** 30 de março de 2026

## Finalidade desta orientação

Este documento define exatamente até onde os grupos devem chegar no
DreamShaper para que a entrega do **Trabalho 1 (T1)** seja considerada
adequada. O T1 não exige execução do projeto nem resultados finais.
Exige que o grupo esteja com o projeto suficientemente estruturado para
**iniciar a coleta de dados**.

## O que o T1 exige

Segundo **orientações do trabalho**, o grupo deve entregar:

- apresentação do problema
- pergunta de pesquisa
- variáveis
- instrumento de coleta pronto para aplicar
- respostas do DreamShaper referentes a essa fase

Portanto, até **30 de março de 2026**, o grupo deve ter concluído no
DreamShaper tudo o que for necessário para sair da fase de ideia e
entrar na fase de coleta.

## Etapas do DreamShaper que devem estar preenchidas até o T1

### 1. `PROJETO DE EXTENSÃO`

Deve conter:

- título provisório ou definitivo do projeto
- identificação clara do tema

Critério de suficiência:

- o título deve indicar claramente sobre o que trata a pesquisa

### 2. `ALUNOS ENVOLVIDOS`

Deve conter:

- nomes dos integrantes do grupo

Critério de suficiência:

- todos os integrantes que participam da entrega devem estar listados

### 3. `DEFINIÇÃO DO PROJETO`

Deve conter:

- apresentação objetiva do problema
- delimitação do recorte
- contexto em que o problema será observado
- pergunta de pesquisa escrita de forma explícita

Critério de suficiência:

- deve ser possível ler essa seção e entender claramente o que será
  investigado
- a pergunta de pesquisa deve aparecer em forma interrogativa ou
  formulação equivalente inequívoca

### 4. `OBJETIVOS`

Deve conter:

- objetivo geral coerente com a pergunta de pesquisa
- objetivos específicos compatíveis com a coleta e análise pretendidas

Critério de suficiência:

- os objetivos precisam derivar do problema e não apenas descrever
  intenções vagas

### 5. `COMUNIDADE: PÚBLICO-ALVO`

Deve conter:

- quem responderá ao instrumento
- quantidade prevista de participantes
- critério básico de escolha dos participantes

Critério de suficiência:

- o público-alvo precisa estar compatível com a pergunta de pesquisa

### 6. `ANÁLISES DAS NECESSIDADES`

Deve conter:

- justificativa curta do recorte escolhido
- explicação de por que o problema merece investigação
- coerência entre o problema e o público-alvo

Critério de suficiência:

- a seção deve justificar a relevância da pesquisa e sustentar a escolha
  do recorte

### 7. `METODOLOGIA`

Esta é a etapa mais importante para o T1.

Deve conter:

- tipo de coleta a ser realizada
- instrumento de pesquisa definido
- variáveis explicitadas
- indicação do tipo/escala das variáveis, quando aplicável
- explicação de como cada variável será observada no instrumento
- questionário ou instrumento de coleta pronto para aplicação

Critério de suficiência:

- a coleta deve poder começar imediatamente após a aprovação da entrega
- o instrumento não pode estar apenas “planejado”; ele deve estar pronto

### 8. `PLANEJAMENTO DO PROJETO`

Deve conter ao menos:

- previsão inicial de aplicação do instrumento
- organização mínima da etapa de coleta

Critério de suficiência:

- o cronograma deve mostrar que o grupo sabe quando e como iniciará a
  coleta

## Onde o link do formulário deve ser incluído

O link de acesso ao formulário ou questionário deve ser incluído
**principalmente na etapa `METODOLOGIA`**, porque é nessa etapa que o
grupo descreve:

- como os dados serão coletados
- qual instrumento será utilizado
- como as variáveis serão operacionalizadas

Essa é a localização mais adequada porque o link faz parte do
**instrumento de coleta**, e o instrumento é um elemento metodológico.

## Recomendação prática sobre o link

Para evitar dúvida na correção, o grupo deve inserir o link em dois
lugares:

### Local obrigatório

`METODOLOGIA`

Inserir:

- nome do instrumento
- breve descrição do instrumento
- link clicável do formulário

Exemplo:

`Instrumento de coleta: Questionário sobre interesse em internacionalização`

`Link do formulário: https://...`

### Local complementar recomendado

`PLANEJAMENTO DO PROJETO`

Inserir:

- referência curta de que a coleta será feita com o formulário já
  disponibilizado

Isso não substitui a presença do link em `METODOLOGIA`, mas ajuda a
mostrar que o grupo está pronto para aplicar.

## Regra de avaliação sobre o link do formulário

Para o T1, o grupo só deve ser considerado **pronto para iniciar a
coleta** se houver evidência concreta do instrumento. Essa evidência
pode ser:

- perguntas transcritas diretamente no DreamShaper
- link funcional para o formulário
- instrumento anexado ou reproduzido de modo completo

Se o grupo apenas mencionar “Google Forms”, “questionário”, “formulário”
ou “entrevista”, mas não apresentar o conteúdo nem o link, então o item
**instrumento de coleta pronto para aplicar** não deve ser considerado
atendido.

## Checklist objetivo para correção

O grupo deve ter, no mínimo:

- problema apresentado com clareza
- pergunta de pesquisa explícita
- variáveis definidas
- tipo/escala das variáveis indicado quando necessário
- público-alvo definido
- justificativa do recorte
- metodologia de coleta descrita
- questionário ou instrumento pronto
- link do formulário em `METODOLOGIA` ou instrumento integralmente
  reproduzido
- planejamento inicial da aplicação

## Síntese final

Até **30 de março de 2026**, o grupo deve estar com o DreamShaper
completo até `METODOLOGIA`, com elementos iniciais de
`PLANEJAMENTO DO PROJETO`, e com o **instrumento de coleta já
acessível**.

**Em termos práticos, isso significa:**

- o projeto já deve estar conceitualmente fechado
- a pergunta de pesquisa já deve estar definida
- as variáveis já devem estar organizadas
- o questionário já deve existir
- o link do formulário deve aparecer preferencialmente em `METODOLOGIA`
